# Contributing to SetFlow.ai

Thank you for your interest in contributing to SetFlow.ai! We welcome contributions from developers, designers, and testers of all skill levels. 

Here is a guide on how you can get started and make contributions to the repository.

## 🚀 How to Contribute

### 1. Reporting Bugs
If you find a bug, please create a GitHub issue with the following details:
- A clear description of the bug.
- Steps to reproduce the behavior.
- Expected vs. actual results.
- Your OS, Node.js version, and any relevant logs.

### 2. Feature Requests
Have an idea for SetFlow.ai? Feel free to open a feature request in the Issues tab describing what you want to add and why it would benefit users.

### 3. Submitting Code Changes (Pull Requests)
If you want to contribute code to fix a bug or add a feature, please follow these steps:
1. **Fork** the repository to your own GitHub account.
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/your-username/setflow-ai.git
   ```
3. Create a new branch for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. Make your code changes and test them locally.
5. Commit your changes with a descriptive commit message:
   ```bash
   git commit -m "feat: add calendar auto-confirm webhook support"
   ```
6. Push the branch to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
7. Open a **Pull Request** (PR) back to the main repository, describing the changes you made and linking any related issues.

## 🛠️ Local Development Setup
- Install dependencies: `npm install`
- Start local development server: `node server.js`
- Standard port is `3000`. Set proxy variables in `.env` if routing requests through local proxies.

## ⚖️ Code of Conduct
Please be respectful and supportive of other contributors. Let's work together to build a helpful, friendly automation tool!
